## О проекте

**TLVanced** — это десктопное приложение для Windows: одно окно-лаунчер и несколько отдельных экранов — **Engine.ini**, **Input.ini**, **GameValue.ini**, **HUD (Main.azj)**. Всё делается локально на машине, без серверов и аккаунтов.

> Проект создан величайшим погромистом и стримером ТАНГЕНСОМ и **никак не связан** с криворуким издателем или разработчиком игры. Используйте на свой страх и риск; перед изменениями имеет смысл сделать резервные копии файлов.

## Возможности

### 1. Треш настройки

Окно с переключателями для исправления идиотских настроек **Throne and Liberty** — файл:

`%LocalAppData%\TL\Saved\Config\WindowsNoEditor\GameValue.ini`

Там же строки вида `TLGame.TLOption....#1=true/false` (Экранный диктор, 3D-звук (В 99% случаях он вам не нужен), Groom-волосы и т.д.). При **каждом** переключении тумблера значения сразу перезаписываются в файл.

**Важно:** если нужной строки ещё нет в `GameValue.ini`, пройдитесь по тумблерам **туда-обратно** (или хотя бы один раз измените состояние) — так приложение гарантированно **допишет** настройки в файл. Пока переключатель ни разу не менялся, соответствующая строка могла не попасть в ini.

---

### 2. Фиксы Ввода

Окно с переключателями для отключения настроек интерполяции мыши, выключение всех делает **мышь более предсказуемой** — правка секции в:

`%LocalAppData%\TL\Saved\Config\WindowsNoEditor\Input.ini`

Используется блок **`[/Script/Engine.InputSettings]`**: сглаживание мыши, масштаб чувствительности от FOV, эмуляция тача мышью и т.п. Значения пишутся как `True`/`False` в стиле UE. Чужие ключи в этой секции не затираются.

**То же правило про тумблеры:** чтобы параметры **точно появились** в `Input.ini` (особенно в «пустом» или старом файле), **покликайте каждый нужный тумблер туда-сюда** — запись выполняется при изменении переключателя, а не «просто открыли окно и закрыли».

---

### 3. UE4 Конфиг

Добавляет и редактирует параметры запуска **Unreal Engine 4** в конфиге клиента **Throne and Liberty** — через файл:

`%LocalAppData%\TL\Saved\Config\WindowsNoEditor\Engine.ini`

В отдельном окне вы получаете **таблицу**: имя cvar, текущее значение (если в конфиге пусто — показывается `*`), подсказки по допустимым значениям и краткое описание. Удобно **оптимизировать FPS**, убирать тяжёлые эффекты и **тонко донастраивать** картинку под своё железо — от motion blur и тумана до Niagara, травы, foliage и атмосферы неба.

- Читается секция **`[/Script/Engine.RendererSettings]`**
- Если секции ещё нет — она будет создана при сохранении.
- Строки, которых нет в вашем конфиге, при сохранении **не затираются** — правятся только известные ключи и добавляются новые.

---

### 4. UI Фикс

Убирает с экрана два навязчивых HUD-элемента — **диалог с NPC** и **диалог с Амитоем** — за счёт правки лейаута Azulejo:

`%UserProfile%\Documents\TL\UserHUD\Main.azj`

Для компонентов с ключами **145** и **128** выставляются: выравнивание **LeftTop**, масштаб **0**, смещение **(-1000, -1000)** — фреймы уезжают за пределы видимой области и перестают мешать.

- Кнопка **красная**, если `Main.azj` не найден.
- Текст кнопки **зелёный**, если оба компонента уже «убраны» по этим правилам.

Чтобы это заработало, нужно экспортировать профиль, который потом будет отредактирован TLVanced
- Заходим в игру
- Открываем HUD Редактор
- Слева внизу жмём кнопку Экспорта (Стрелочка)
- В окне ввода названия пишем - Main! ОБЯЗАТЕЛЬНО!
- В TLVanced нажимаем UI Фикс
- Если надпись загорелась зелёным, значит всё окей!
- В игре, в HUD Редакторе нажимаем кнопку Изменить
- Справа от названия файла будет стрелка
- Жмём загрузить
- Кайфуем

## Сборка

```bash
git clone https://github.com/tnhgnz/TL-VANCED.git
cd TLVanced/TLVanced
dotnet restore
dotnet build -c Release
```

Готовый exe появится в `TLVanced/bin/<Platform>/Release/net9.0-windows/`.

> В репозитории используется локальная ссылка на **Wpf.Ui** (`Libs\Wpf.Ui.dll`) — убедитесь, что файл есть в дереве проекта, либо замените ссылку на NuGet-пакет под вашу среду.
